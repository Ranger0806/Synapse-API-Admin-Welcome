# Synapse API Admin Welcome

CLI-утилита для отправки **server notice** пользователям Synapse через Admin API.

## Возможности

- Режим `-a`: отправка только новым пользователям.
- Режим `-n`: отправка всем пользователям из локальной SQLite БД.
- YAML-конфиг (`config.yml`) вместо переменных окружения.
- YAML-шаблон сообщения (`body` + `formated_body`).
- Логирование одновременно в `logs.log` и в консоль.

## Стек

- Python 3.11+
- `httpx` для HTTP-запросов
- `aiosqlite` для локальной БД
- `PyYAML` для YAML-конфигов

## Установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Конфигурация

Создайте или отредактируйте `config.yml`:

```yaml
homeserver_url: "https://your-synapse.example"
access_token: "replace-with-admin-token"
db_path: "users.db"
```

### Пояснения полей

- `homeserver_url` - базовый URL Synapse (без завершающего `/`).
- `access_token` - админский токен для Synapse Admin API.
- `db_path` - путь к SQLite-файлу с пользователями.

## Файл сообщения

Пример `message.yml`:

```yaml
body: "Привет! Это уведомление от администратора."
formated_body: "<p><strong>Привет!</strong> Это уведомление от администратора.</p>"
```

Поддерживаются ключи:

- `body` (обязательный)
- `formated_body` (рекомендуемый)
- `formatted_body` (поддерживается для совместимости)

Если HTML-поле не указано, берется значение `body`.

## Запуск

### 1) Инициализация БД и отправка только новым (`-a`)

```powershell
python -m main -a message.yml
```

Поведение:

- Если БД еще нет: сохранит текущих пользователей и завершится без рассылки.
- Если БД есть: отправит сообщение только пользователям, которых еще нет в БД.

### 2) Отправка всем пользователям из БД (`-n`)

```powershell
python -m main -n message.yml
```

Поведение:

- Если БД не найдена: завершится с кодом `2`.
- Иначе отправит сообщение всем пользователям из БД.

## Логи

Логи пишутся одновременно:

- в файл `logs.log`
- в консоль

Формат записи:

- время
- уровень (`INFO`, `WARNING`, `ERROR`)
- имя логгера
- сообщение

## Коды возврата

- `0` - успешное выполнение
- `1` - внутренний fallback (невыбранный режим, практически недостижим)
- `2` - режим `-n` запущен без существующей БД

## Структура проекта

```text

message.yml     # шаблон сообщения
logs.log        # runtime-логи
```
